# Kongtze - Project Vision and Requirements

## Project Overview
Kongtze is an AI-powered education app with study planning and gamified testing for a 10-year-old student.

## 1. Basic Features

### 1.1 Study Calendar Planning
- **Weekly basis** calendar system
- **2-3 sessions per day**
- Each session: **~30 minutes**
- Different subjects for each session

### 1.2 Subjects Covered
- Math
- English
- Chinese
- Science
- (Extensible for additional subjects)

### 1.3 Test Generation Types
1. **School Homework Tests**
   - Upload test photos
   - Generate testing interface from uploaded images
   
2. **Extra Practice Tests**
   - Additional testing for the student
   - Generated by the system

### 1.4 AI-Enabled Features
- **Dynamic Difficulty Adjustment**
  - Adjusts based on student's test results
  - Personalized to performance level
  
- **Time Limits per Question**
  - AI-generated based on:
    - Question difficulty level
    - Student's previous performance in that category
  - Real-time countdown during test
  
- **Question Explanations**
  - Available after test completion
  - Helps student learn from mistakes

### 1.5 Test Results & Reporting
- **End-of-test report** includes:
  - Overall score
  - Per-question breakdown:
    - Score for each question
    - Right/wrong status
  - Option to generate explanations for each question

## 2. Gamification System

### 2.1 Scoring System
- Each question has a score
- Each test has a total score
- Real-time score calculation based on student input

### 2.2 Rewards System
- **Daily Performance Tracking**
  - Earn reward points based on daily scores
  - Accumulative points system

### 2.3 Rewards Exchange
- Points can be exchanged for gifts
- **Lucky Draw System**
  - Blind box draws per session
  - Kid-friendly reward mechanism

## 3. Target Users

### 3.1 Primary User - Student
- **Age:** 10 years old
- **School:** Year 5, Dulwich College Singapore
- **Academic Profile:**
  - **Math:** AA Class group (most advanced), but struggling
  - **Chinese:** Weak in writing
  - **English:** Average performance
- **Note:** App is customized specifically for this student

### 3.2 Secondary User - Parent
**Parent Interface Features:**
- Set up subjects for weekly calendar
- Modify and adjust generated calendar
- **Dashboard** showing:
  - Student's performance metrics
  - Progress tracking
  - Test results overview

## 4. Technical Approach

### Development Sequence (Recommended):
1. **Understand full scope** of the application
2. **Design Data Structure** and Database schema first
3. **Design Backend API Structure** to support frontend needs
4. **Develop Frontend** with kid-friendly, stylish UI

### Design Requirements:
- **Frontend:** Kid-friendly interface, appealing to 10-year-olds
- **Style:** Colorful, engaging, age-appropriate design

## Key Business Rules & Constraints
- Single-user system (for one specific student)
- Focus on Singapore education system (Dulwich College)
- Must support photo upload for homework tests
- AI must adapt to individual learning pace
- Gamification to maintain engagement and motivation
